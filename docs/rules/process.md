# 研究論文執筆プロセス (ResearchProject_Trial1)

**コンテキスト:** アカデミック学術論文（認知科学 × 制御工学）
**ゴール:** 国際会議採択レベルの品質を目指す。
**緊急目標:** **本日中にドラフトを完成させる**（超短期集中）。

## 1. 基本戦略: "Top-Down Logic First"
時間が限られているため、文章の下書きから始めず、**論理構造（アウトライン）の完成**を最優先する。論理が通っていれば、文章化はAIと連携して高速に行える。

## 2. ワークフロー: "Rapid Academic Sprint"

### Phase 1: 構造化 (Structure) - [目安: 30分]
1.  **Core Contribution:** 論文の「核」となる新規性を1文で定義する。
2.  **Storyline:** 問題提起 -> 提案手法 -> 検証結果 -> 結論の流れを箇条書きで確定する。
3.  **Outline:** セクションごとの主な主張と、配置する図表を決定する。

### Phase 2: コンテンツ生成 (Drafting) - [目安: 数時間]
各セクションを以下の順序で埋める（書きやすい順ではない、論理の要となる順）。
1.  **Method:** 提案する理論やモデルの記述（数式、アルゴリズム）。
2.  **Results/Experiments:** 検証方法と（想定される）結果。
3.  **Introduction:** 背景、課題、解決策、貢献の提示。
4.  **Related Work:** 既存研究との差異化（認知科学と制御工学の交差点）。
5.  **Conclusion & Abstract:** 全体の要約。

### Phase 3: 品質向上 (Refining) - [随時]
- **Three-Strike Rule (Research版):** 主張に論理的飛躍がないか、3つの視点（査読者、競合研究者、初学者）から批判的にチェックする。
- **Citation Check:** 重要な参考文献が漏れていないか確認する。

## 3. ドキュメント標準
- **形式:** LaTeX または Markdown (最終的にPDF化)。
- **言語:** 英語（国際会議ターゲット）または日本語（まずはロジック構築のため）。※ユーザー指示に従う。
- **図表:** `docs/images/` に配置。概念図はMermaidまたは生成AIで即座に可視化する。

## 4. 品質ゲート (Target: International Conference)
少なくとも以下をクリアすること：
-  **Novelty:** 既存研究との差分が明確か？
-  **Clarity:** イントロダクションだけで貢献が理解できるか？
-  **Validity:** 提案手法の妥当性が（論理的あるいは実験的に）示されているか？
